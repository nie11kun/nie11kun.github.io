使用 frp 实现内网穿透

  • 作者: admin
  • 时间: 2019-11-25
  • 分类: web

对于大多数家庭网络用户，都没有独立 IP 地址使用，在本地搭建网络服务无法直接在外网访问，使用 frp 可以实现这一需求。

项目地址：https://github.com/fatedier/frp
使用手册：https://github.com/fatedier/frp/blob/master/README_zh.md

环境需求：
一台有独立 IP 的服务器
可安装 frp 的本地主机或路由器(windows/linux)

安装

从 release 页面分别下载合适版本的最新程序到内网主机和独立 IP 的服务器。

程序内包含服务端程序(frps)及客户端程序(frpc)，以及对应配置文件。

配置文件

服务端 frps.ini 示例：

[common]
token=464116963
bind_port = 7000
kcp_bind_port = 7000
vhost_http_port = 8080
allow_ports = 6000

客户端 frpc.ini 示例：

[common]
token=464116963
server_addr = xxx.xxx.xxx.xxx
server_port = 7000
tls_enable = true
use_compression = true
protocol = kcp

[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 6000

[web]
type = http
local_port = 80
custom_domains = domain/IP
http_user = admin
http_pwd = xxxxxxxxxxx

token：服务端和客户端的 common 配置中的 token 参数一致则身份验证通过，可以不写此配置
server_addr：远程独立 IP 地址
bind_port 设置同 server_port
allow_ports：为给客户端开放的端口，用于和客户端数据交互，如不设置此条目，则默认开放所有端口
vhost_http_port：web 服务访问端口，如：test.com:8080，即可访问内网 web 页面
如访问 web 页面是不需要密码则去掉 客户端 http_user 及 http_pwd

详细配置文件：
frps：https://github.com/fatedier/frp/blob/master/conf/frps_full.ini
frpc：https://github.com/fatedier/frp/blob/master/conf/frpc_full.ini

启动服务

服务端：

./frps -c ./frps.ini

客户端：

./frpc -c ./frpc.ini

测试

访问内网 ssh

ssh -oPort=6000 [email protected]

IP 为远程服务器的 IP，username 为内网设备的用户名。输入后提示密码，也为内网设备 ssh 密码。

web 服务

http://www.yourdomain.com:8080

如果设置有用户名密码，会提示口令，然后进入 web 页面。

以上就是 frp 的简单使用，高级用法参考官方手册。
